Introduction

The purpose of this application is to create a virtual version of Monopoly. The game will be playable by 1-4 players at a time. Computer players are not supported. The application will replicate the aspects of the real-life Monopoly game and be able to be played by anyone with a Windows 10/11 operating system and with Python 3.11x and Python Arcade installed. This document will provide instructions on how to install the game, as well as a detailed outline of how the flow of gameplay will progress. It will serve as a user guide to assist players as they navigate the game. This document will also provide requirements for the game and how each aspect is implemented in its creation.

Installation

Option 1: Portable

The game can optionally be run without installation by downloading and running the “Monopoly.exe” variant directly. The user is free to place this executable anywhere they like and no permanent files are created after running that need to be cleaned up when the user wishes to remove the game. To remove, simply delete Monopoly.exe.

Note: the portable version of the game requires referencing the online version of the user guide as no other files are distributed with this version.

Option 2: Installer Package

Monopoly-Setup.exe can be downloaded and run as an installation wizard for the game executable as well as user guide. The wizard offers several user selectable customization options around making the installed game easier to access. The installer package also offers standard uninstall functionality to remove all assets when the user wishes to remove the game.

Running the Installer Package

Step 1: On running Monopoly-Setup.exe the user will be prompted to select whether to install the game for just their account or all users on the computer.

Note: Installing for all users requires the installer to be given administrative privileges.

Step 2: The license agreement is presented and the user must accept the agreement to continue installation.

Step 3: The installer presents the default install location for the chosen install method.

The user is able to override this by either typing a path or graphically browsing to noe with the “browse” button:

Step 4: The user can optionally have a desktop shortcut to the game be placed on their desktop.

Step 5: The installer will review the installation location and prompt the user for confirmation one last time before performing the actions.

Step 6: The progress and current step of installation is shown to the user.

Step 7: Installation is complete! By default a box to run the game when the user clicks “finish” is checked. The user can optionally uncheck this if they do not wish to run the game right away.

Uninstalling Monopoly

The installation package will create a program entry which can launch the uninstaller from the standard system application management view.

Note: The portable version (Monopoly.exe) does not create or require an uninstaller. It can simply be deleted to be removed completely.

Once the program is located, click the “...” menu and locate the uninstall button to launch the uninstallation wizard:

Gameplay

Home Screen

The home screen is the landing page when the application is first opened. It gives the options to Start, view the help, or get exit. To begin setting up the game, the player will need to press Start.

Player Registration

Once the Start button is pressed, the page to register players will be shown. When the “Enabled” button is shown, the player will be included in the game. The names of each individual player are able to be edited and the player piece color can also be changed.

Board

The board represents the main playspace for the game. Each player’s selected piece is placed on the board. As the game is played the pieces are moved around the board. Actions are triggered based on the spaces players land on.

To the right of the board is the current game status:

This includes a tally of each player’s current cash and the current space the player is on. The current player’s turn is written in blue. The “Roll Dice” button rolls the dice, displaying the value and automatically moving the player.

Player Turn

At the beginning of each player’s turn, players will choose any of the below turn based actions to perform. If the user has no other actions to perform they will roll the dice to advance their turn. The dice will show how many spaces to move and automatically move the player to the correct position. What happens then depends on the spaces the user moves past and lands on, as outlined below

Buying Property

When a player lands on a property, they will be given the option to buy or reject. The player will only be allowed to buy the property if they have enough money to do so. If the player chooses not to buy the property, their turn will end and the game will continue on.

Buying Houses/Hotels

Players which own all spaces of a given color are able to buy houses at the start of their turn. Up to 4 houses can be bought on each property, at which point a final upgrade to a hotel can be purchased. The more upgrades, the higher the factor of rent from the base value.

Mortgaging Property

Once a player owns a property, they will have the option to mortgage it during each turn. If they choose to follow through with the mortgage, they will receive the money that the property is worth and continue on with their turn. If a property is mortgaged, they will have the option to buy it back each turn, given they have enough money to do so.

Unique Spaces

Several spaces on the board offer unique events. Community Chest and Chance spots result in triggering a draw from those card piles on the board. Each card drawn defines the resulting actions.

Free Parking, GO, and Visiting offer “safe” spaces to land on which do not trigger any resulting actions. Passing go results in getting the amount of money defined on the space.

“Go To Jail” results in the player being sent to jail without collecting money for passing Go. If a player has a get out of jail free card they may leave jail immediately. A player is able to either roll doubles within 3 tries or pay $50 to get out of jail. If the player has rolled 3 times without doubles they must pay $50. Once out of jail their piece moves to the “Just Visiting” portion.

Ending the game

The game ends when all but one player has gone bankrupt. The remaining player is declared the winner of the game.

Details

Python 3.11x

Python 3.11x is the Python version that’s used to code and run this application. Python Arcade, the platform the game is built through, doesn’t support newer versions of Python. Python 3.11x will need to be installed in order for the successful playing of the game from source. For prebuilt .exe versions of the game, no Python installation is required

Python Arcade

As previously noted, Python Arcade is the platform where the game is able to be run through and actually played on. It must also be installed in order to successfully play. Python Arcade allows for the game graphics to be viewed and the audio files to be heard while the game is running.

Windows 10/11 Operating System (64 bit)

A computer with a Windows 10/11 operating system (64 bit) is necessary to run the game. This is due to convenience and to ease the building process because every member of the group has a computer running Windows and will be able to access the game.

Virtual Monopoly Board

The virtual Monopoly board is a pivotal part of the game because it is where the actual gameplay is visualized. The board, uploaded as a png file, is identical to the real-life classic Monopoly board. The player pieces will be able to move around and houses and hotels will be added as gameplay progresses.

Moveable Player Pieces

The player pieces are also one of the most important aspects of the game, as they show where each player is located. Upon complete implementation, they will be able to move around the board in accordance with the numbers shown on that dice during a given player’s turn.

Drawable Cards

Game cards are also one of the most important parts of our virtual game, as they are included in the real-life Monopoly game and are drawn when players land on certain spots. The cards in our application will be able to be drawn and will have actions on them that also correspond to the real-life game. The two types of cards that will be included are Community Chest and Chance.

Virtual Houses and Hotels

Similar to cards and player pieces, virtual houses and hotels will be added into the game that increase the value of the properties. These are also part of the actual Monopoly game and will be able to be bought when a player meets the criteria (i.e. owns all of the properties of the same color).

Pair of Dice

A pair of dice is necessary for this game so that players can roll them and move around the board. It’s important that the dice are randomized as they would be in real-life. When it becomes a player’s turn, their first action will be to roll the dice. Players will be able to roll again if they roll doubles. If three doubles are rolled in a row, they will be sent to jail, as outlined in the Monopoly instructions.

It then covers walking the user through the menu system and conditions for setting up a new game. The full game rules of Monopoly are also covered, along with in-game illustrations where appropriate.

Incomplete:

Given more time these elements could be tied into the UI but the game right now presents a configurable and documented base version of monopoly where the players are able to move around the board and collect $200 for passing go.
